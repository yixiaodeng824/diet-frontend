# 膳食健康助手 (Dietary Health Assistant) — 项目工作总结

> 微信小程序 + Flask 后端，通过拍照识别食物、追踪每日营养摄入，并根据个人健康目标提供 AI 驱动的饮食建议。

---

## 一、项目架构

```
大创前端/          ← 微信小程序（客户端）
大创后端/          ← Flask Python 服务（服务端）
d:/
 └── food-101/    ← Food-101 图像数据集（101 类食物，约 101,000 张图）
```

### 技术栈总览

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | 微信原生小程序 (WXML/WXSS/JS) | AppID: `wx4486a9ebe8d3e79b`, SDK 3.14.1 |
| 后端 | Flask 2.3.3 + PyTorch 2.x | CORS 跨域，RESTful API |
| 模型 | Ultralytics YOLOv8l-cls | 在 Food-101 上微调，101 类食物识别 |
| 数据库 | SQLite + JSON | user_info.db / food_record.db / nutrition_db.json |
| 训练 | 自定义 LoRA+ 策略 | 余弦退火 + 早停 + 断点续训 |

### 3.3 DeepSeek 联网搜索集成（新增）

在工作流中引入 **DeepSeek API**（兼容 OpenAI SDK），实现 YOLOv8 识别后的智能增强：

#### 工作流

```
用户上传图片
    ↓
YOLOv8l 分类 → 识别食物名称（如"北京烤鸭"）
    ↓
本地 JSON 营养库 → 基础营养数据（热量/蛋白质/碳水/脂肪）
    ↓
DeepSeek API（extra_body: {enable_search: true}）→ 联网搜索
    ↓
返回结构化百科：简介 / 营养功效 / 健康建议 / 做法推荐 / 搭配建议
```

#### 核心组件

| 文件 | 作用 |
|------|------|
| `app/services/deepseek_service.py` | 封装 DeepSeek API 调用，支持普通查询和流式查询 |
| `app/routes/deepseek.py` | 两个新端点：`/detect/deepseek`（非流式）、`/detect/deepseek/stream`（SSE 流式） |
| `.env.example` | 环境变量模板（`DEEPSEEK_API_KEY`） |

#### 关键实现

- 使用 **OpenAI 兼容客户端** (`openai.OpenAI`)，base_url 指向 `https://api.deepseek.com`
- 通过 `extra_body={"enable_search": True}` 开启 DeepSeek 联网搜索能力
- 将本地营养数据作为上下文注入 prompt，AI 生成信息更精准
- 未配置 API Key 时自动降级，仅返回 YOLO 识别结果（不报错崩溃）

#### 前端接入方式

**非流式**（简单调用）：
```js
wx.uploadFile({
  url: 'http://192.168.31.167:5000/api/detect/deepseek',
  filePath: tempFilePath,
  name: 'image',
  success(res) {
    const data = JSON.parse(res.data);
    // data.yolo_result      → YOLO 识别结果
    // data.deepseek_result  → AI 百科信息
  }
});
```

**流式**（打字机效果）：
```js
const resp = await fetch('/api/detect/deepseek/stream', { method:'POST', body: formData });
const reader = resp.body.getReader();
// 按 SSE 格式逐块读取 data.<event> 事件
```

### 3.4 食物识别模型

- **基座模型：** `yolov8l-cls.pt`（Ultralytics YOLOv8 Large，分类头）
- **训练数据：** Food-101 数据集（101 类，约 101,000 张图片）
- **预处理：** Resize(224) → CenterCrop(224) → TorchVision 归一化
- **推理增强：** 锐化 + 对比度增强（改善低质量图片识别效果）
- **标签：** 101 类英文 → 中文完整映射表

### 3.5 训练策略（LoRA+）

`train_yolo.py` 支持三种训练模式：

1. **全量微调 (full)** — 直接调用 Ultralytics 训练
2. **分阶段微调 (freeze_then_unfreeze)** — 先冻结主干训练分类头，再解冻全量训练
3. **LoRA+ (lora_plus)** — 自定义低秩适配器，对 down/up 投影使用不同学习率（当前主力方案）

LoRA+ 训练特性：
- 分类器输出从 1000 类替换为 101 类
- 分类器头部全量更新（不被 LoRA 冻结）
- 余弦退火学习率调度（断点续训恢复 T_max）
- 早停 + 断点自动保存
- 训练历史 CSV/JSON + 曲线图
- 数据增强：随机裁剪、翻转、颜色抖动
- **当前最佳：** 78 轮后验证准确率 ≈ **79%**

## 四、数据与资源

### 4.1 Food-101 数据集 (`d:\food-101\`)

- **来源：** 标准 Food-101 学术数据集
- **规模：** 101 个类别 × 约 1000 张/类 = ~101,000 张图片
- **划分：** 训练集（75%）+ 测试集（25%），官方 JSON 元数据
- **用途：** 用于训练 YOLO 分类模型

### 4.2 营养数据库

- **文件：** `app/data/nutrition_db.json`
- **内容：** 101 种食物每 100g 的营养成分（热量、蛋白质、碳水、脂肪）
- **用途：** 检测识别后，按食物名称查询营养数据并返回给前端

### 4.3 数据库

| 数据库 | 表 | 字段 | 用途 |
|--------|-----|------|------|
| `user_info.db` | `user_info` | user_id, height, weight, goal, name, create_time, update_time | 用户配置 |
| `food_record.db` | `food_record` | id, user_id, food_name, weight, calories, protein, carbs, fat, record_time | 饮食记录 |
